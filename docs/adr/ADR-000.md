# ADR-000 — Convenção de numeração de ADR

- **Status:** accepted
- **Data:** —

## Contexto

ADRs precisam de IDs estáveis e citáveis. Num projeto de um repo só, basta uma sequência. Mas se o projeto crescer pra vários repos que se referenciam, numeração local colide (dois repos teriam um "ADR-007" diferente).

## Decisão

**Enquanto for single-repo:** sequência simples `ADR-NNN`, sem prefixo, começando em `ADR-001`. (`ADR-000` é este meta-ADR.)

**Se virar multi-repo:** cada repo ganha um **prefixo curto próprio** e mantém sequência independente:

- repo incumbente (o primeiro): continua `ADR-NNN` sem prefixo — não retrofitar.
- repos novos: prefixo + sequência própria começando em 001, ex.:
  - `sdk` → `SDK-ADR-001`, `SDK-ADR-002`, …
  - `app` → `APP-ADR-001`, …
- registre o prefixo de cada repo novo **aqui** antes do primeiro uso.

**Referência cruzada sempre com prefixo completo.** Ao citar um ADR de outro repo, use o prefixo (`SDK-ADR-002`). Bare `ADR-NNN` significa sempre o repo incumbente. Nunca cite bare querendo dizer outro repo.

## Consequências

IDs estáveis desde o dia 1; a transição mono → multi não exige renumerar nada (o incumbente fica sem prefixo). Custo: ao escalar, é preciso lembrar de registrar o prefixo de cada repo novo aqui. Single-repo não paga nada por isso.
