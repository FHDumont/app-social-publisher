# ADR-001 — Persistência em SQLite, não Postgres

- **Status:** accepted
- **Data:** 2025-02-03
- **Fase:** F-base

## Contexto

Recados é uma API de notas self-hosted, single-user, rodando local. Precisa persistir notas. As opções óbvias eram SQLite (embarcado, arquivo único) e Postgres (servidor separado).

## Decisão

Usar SQLite, arquivo único no volume do projeto.

## Alternativas consideradas

- **Postgres** — robusto e familiar, mas exige um serviço separado pra rodar, configurar e fazer backup. Peso desproporcional pra um app single-user local.

## Consequências

Setup trivial (sem serviço extra), backup é copiar um arquivo, deploy é mais simples.

Custo: escrita concorrente é limitada e não há acesso por rede ao banco — ambos irrelevantes no uso single-user local. Se um dia o app virar multi-user ou precisar de acesso remoto ao dado, isto vira candidato a um novo ADR que supersede este.
